# Python for DevOps — Course Repository

## Directory Structure

```
python-dev-course/
├── class1/ … class12/      Student-facing folders (slides, quiz, homework)
├── Exams/                  Instructor-private workspace
├── Slides_Analysis/        Tooling to read .pptx speaker notes
└── .github/workflows/      GitHub Actions (Windows .exe builder)
```

Each `classN/` folder is what students receive:

| File | Description |
|------|-------------|
| `slidesN.pptx` | Lecture slides |
| `quizN` / `quizN.exe` | Compiled quiz executable (added after building) |
| `homeworkN.ipynb` | Homework notebook (added when ready) |

## Workflow: New Class

### Step 1 — Extract speaker notes from your slides

Run this from `Slides_Analysis/` to read the code examples and notes you wrote in PowerPoint:

```bash
cd Slides_Analysis

# Print slide body text + speaker notes to stdout
.venv/bin/python extract_notes.py ../classN/slidesN.pptx

# Speaker notes only (faster reading when designing quiz questions)
.venv/bin/python extract_notes.py ../classN/slidesN.pptx --notes-only

# Write to a file for offline review
.venv/bin/python extract_notes.py ../classN/slidesN.pptx -o classN_notes.txt
```

`Slides_Analysis/.venv` is set up once and reused for all classes. If it is ever missing:

```bash
cd Slides_Analysis
python3 -m venv .venv
.venv/bin/pip install python-pptx
```

### Step 2 — Create the quiz JSON

Create `Exams/cN.json` based on what you see in the notes. The format is:

```json
{
  "quiz_title": "Python for DevOps — Class N",
  "questions": [
    {
      "question": "What does this print?",
      "code": "x = 10\nprint(x)",
      "options": ["9", "10", "11", "Error"],
      "correct": 1,
      "explanation": "x was assigned 10, so print(x) outputs 10."
    }
  ]
}
```

| Field | Required | Notes |
|-------|----------|-------|
| `quiz_title` | Yes | Shown in the window title bar |
| `question` | Yes | Plain text question |
| `code` | No | If present, shown in a monospace green-on-dark code box |
| `options` | Yes | Exactly 4 strings (A, B, C, D) |
| `correct` | Yes | Zero-based index of the correct option (0=A, 1=B, 2=C, 3=D) |
| `explanation` | No | Shown after the student answers; explains the reasoning |

### Step 3 — Test the quiz without compiling

Copy the JSON as `questions.json` into `Exams/` and run the engine directly:

```bash
cd Exams
cp cN.json questions.json
.venv/bin/python quiz_engine.py
rm questions.json
```

This opens the full GUI immediately so you can check every question without waiting for a PyInstaller build.

### Step 4 — Build the Linux executable

Run from `Exams/`:

```bash
cd Exams

./build_quiz.sh N          # build a single quiz
./build_quiz.sh 1 2 3      # build several at once
./build_quiz.sh all        # build quiz1 through quiz12
```

The finished executable is written to `Exams/dist/quizN`. Copy it to the class folder when ready:

```bash
cp Exams/dist/quiz1 class1/
```

`Exams/.venv` is created automatically on first run if it does not exist. It contains only `customtkinter` and `pyinstaller`.

### Step 5 — Build the Windows executable (.exe)

PyInstaller cannot cross-compile. There are two options:

**Option A — Run on a Windows machine:**

```bat
cd Exams
build_quiz.bat N
```

The `.exe` is written to `Exams\dist\quizN.exe`.

**Option B — GitHub Actions (no Windows machine needed):**

Push your changes. The workflow at `.github/workflows/build-quizzes.yml` triggers automatically when any `Exams/c*.json` or `Exams/quiz_engine.py` changes. It can also be triggered manually:

1. Go to **Actions → Build Windows Quiz Executables → Run workflow**
2. Download the `windows-quizzes` artifact from the completed run
3. Copy the `.exe` files to their `classN/` folders

## What Gets Committed vs Ignored

`.gitignore` excludes all generated and environment files:

| Ignored | Reason |
|---------|--------|
| `Exams/.venv/` | Regenerated by `build_quiz.sh` on first run |
| `Slides_Analysis/.venv/` | Regenerated with `python3 -m venv .venv && .venv/bin/pip install python-pptx` |
| `Exams/build/` | Intermediate PyInstaller artefacts |
| `Exams/dist/` | Compiled executables (copied to `classN/` manually) |
| `Exams/*.spec` | Auto-generated PyInstaller spec files |
| `__pycache__/`, `*.pyc` | Python bytecode |

Everything else is committed: source code, JSON quiz data, build scripts, slides, and `.gitkeep` files that hold the empty `classN/` directories in git.

## Key Files

| File | Purpose |
|------|---------|
| `Exams/quiz_engine.py` | CustomTkinter GUI app; reads `questions.json` from its own directory (dev) or from the PyInstaller bundle (compiled) |
| `Exams/cN.json` | Quiz data for class N — the only file that changes per quiz |
| `Exams/build_quiz.sh` | Linux build script |
| `Exams/build_quiz.bat` | Windows build script |
| `Slides_Analysis/extract_notes.py` | Reads any `.pptx` and prints slide text + speaker notes |
| `.github/workflows/build-quizzes.yml` | Builds all Windows `.exe` files on a GitHub runner |
